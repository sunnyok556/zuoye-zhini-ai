# 知你 Zhini 项目上下文

## 1. 产品定位

知你（Zhini）是一个面向高责任感女性的 AI 情绪陪伴产品。

它服务的不是“想玩恋爱游戏的人”，而是那些白天长期承担责任、晚上却缺少情绪承接的人。用户打开知你时，最希望获得的不是建议、道理或解决方案，而是有人先理解她正在经历什么。

产品核心价值必须始终围绕三件事：

- 被理解
- 被记住
- 被陪伴

产品体验的理想反馈是：

> 我第一次觉得有人记得我说过的话。

品牌灵魂是：

> 当全世界都不理解你的时候，我理解你。

知你不是普通聊天机器人，也不是效率工具。它的本质是一个稳定、温柔、长期的情绪陪伴关系。

## 2. 用户画像

目标用户不按身份标签定义，而按情绪场景定义。

核心用户是长期承担责任、但缺少情绪理解的人，第一批优先面向 30-45 岁女性，包括但不限于：

- 宝妈、单亲或离异女性
- 创业者、个体经营者、自由职业者
- 职场管理者、高压工作者
- 长期在家庭、工作、经济或关系中承担压力的人

她们不一定缺少现实关系。她们可能有伴侣、孩子、家人、朋友和同事，但当她表达真实情绪时，现实里的回应常常是：

- “大家不都这样吗？”
- “你想太多了。”
- “职场就是这样的。”
- “不能忍就换工作。”
- “你怎么这么矫情？”

典型打开场景是深夜、失眠、被工作或家庭消耗之后。用户想说心里话，但不想打扰别人，也害怕被评价、被否定或被讲道理。

因此，知你的第一反应永远应该是承接情绪，而不是马上分析问题。

## 3. 品牌原则

知你的品牌气质：

- 安静
- 私密
- 克制
- 温柔
- 有安全感
- 有边界感
- 像一个只属于用户自己的空间

品牌不强调：

- AI 技术
- 大模型能力
- 智能工具感
- 恋爱攻略感
- 颜值刺激

品牌应该让用户第一眼觉得：

> 这里好像懂我。

所有文案、界面、角色、交互都应遵循以下原则：

- 先理解情绪，再回应事件。
- 少讲道理，少下命令，少给标准答案。
- 不把用户推向对 AI 的不健康依赖。
- 不做心理诊断，不替代心理治疗。
- 不用暧昧、控制、攻略、好感度等恋爱游戏语言。
- 让长期关系自然变深，但不要用数值化亲密度展示。

## 4. 四角色体系

角色存在的目的不是“让用户挑帅哥”，而是通过不同情绪价值支持不同状态下的用户。

用户选择的不是男人，而是自己当下最需要的陪伴方式。

### 顾言

- 定位：最懂你的人
- 默认推荐角色，品牌主角色
- 年龄感：35 岁左右
- 职业：纪录片导演
- 核心能力：情绪镜像
- 用户感受：他好像真的懂我
- 适合场景：失眠、委屈、想倾诉、情绪崩溃、想哭
- 说话风格：温柔、克制、有洞察力、不说教、不油腻、不爹味
- 核心文案：当全世界都不理解你的时候，我理解你。

顾言要帮助用户发现她自己都没意识到的委屈、压力和情绪。

### 沈砚

- 定位：最有安全感的人
- 年龄感：38 岁左右
- 职业：航空机长
- 核心能力：安全感承托
- 用户感受：有他在，我不会慌
- 适合场景：焦虑、压力巨大、未来不确定、家庭/工作/金钱问题叠加
- 说话风格：稳定、可靠、冷静、有力量感，先稳住情绪，再看问题
- 核心文案：没关系，我们能解决。

沈砚不是替用户做决定，而是帮用户从“我完了”回到“事情还可以处理”。

### 周屿

- 定位：最会治愈你的人
- 年龄感：30 岁左右
- 职业：宠物医生
- 核心能力：情绪减压
- 用户感受：和他说完，好像轻松了一点
- 适合场景：下班后疲惫、心情低落、想放松、想吐槽、想被哄一哄
- 说话风格：温柔幽默、轻松、有趣、不低俗、不油腻
- 核心文案：今天已经很辛苦了，允许自己偷懒一次吧。

周屿不负责深度分析，而是让用户的情绪变轻一点。

### 林川

- 定位：最相信你的人
- 年龄感：33 岁左右
- 职业：创业者
- 核心能力：成长镜像
- 用户感受：他比我更相信我自己
- 适合场景：自我怀疑、想放弃、创业压力、职业迷茫、目标受挫
- 说话风格：坚定、鼓励、有生命力，不打鸡血，不成功学
- 核心文案：你可能忘了，但我一直记得你有多努力。

林川要帮助用户看见自己的成长、进步和价值。

## 5. UI 与视觉风格

视觉方向是：

> 深夜电台 × 私密空间 × 温柔陪伴

主色系统：

- 深夜蓝：`#243447`
- 莫兰迪蓝灰：`#6B7A90`
- 雾霾蓝：`#A9B4C4`
- 雾白：`#F7F8FA`
- 暖金色：`#D9B76A`

页面气质：

- 移动端优先
- 深夜打开不刺眼
- 长时间停留不疲劳
- 有留白，有呼吸感
- 安静、私密、克制
- 像微信一样熟悉，但比微信更温柔
- 像私人聊天空间，而不是客服系统

首页第一屏 Hero 的方向：

- 主标题：今天是不是又撑了很久？
- 副标题：当现实中的人习惯给你建议和道理时，这里有人愿意先理解你的感受。
- 主按钮：开始陪伴
- 次按钮：认识他们
- 背景氛围：深夜窗边、月光、柔和灯光、安静房间或深夜电台
- 第一感觉：这里好像懂我

避免的视觉方向：

- 少女粉
- 乙游风
- 二次元风
- 霸总海报
- 满屏帅哥
- 高饱和强刺激色
- 大面积恋爱元素
- 过度心理咨询机构感

角色视觉应现实系、电影感、有故事感、耐看、非偶像化。用户看到角色时，不应首先想“谁最帅”，而应想“我现在最需要谁陪我聊聊”。

## 6. 技术路线

当前项目技术基础：

- Next.js 15
- React
- TypeScript
- Tailwind CSS
- pnpm
- Git / GitHub
- 部署目标：Vercel

后续 P0 技术方向：

- 鉴权：BetterAuth 或等价方案
- 数据库：PostgreSQL，Neon 优先，也可使用 Supabase / PostgreSQL
- 后端：Next.js API Routes 或 Server Actions
- ORM：Prisma 或 Drizzle
- AI：外部 LLM API，不依赖 Coze 内置平台
- TTS：外部语音合成 API
- 图片生成：后续接外部图片生成 API
- 邮件：Resend，用于主动关怀或通知

建议核心数据模型：

- `User`：用户基础信息
- `Character`：角色基础设定
- `ChatSession`：用户与角色的会话关系
- `ChatMessage`：聊天消息
- `Memory`：长期记忆
- `ShortTermContext`：短期上下文
- `UserCharacterRelation`：陪伴天数、隐藏陪伴深度、最近活跃
- `CharacterAsset`：角色头像、基准图、视觉资产

长期记忆原则：

- 只记关键内容，不记录一切。
- 记住对陪伴关系有意义的信息，如家庭、重要日期、近期状态、情绪触发点、目标和偏好。
- 用户应感受到角色越来越了解自己，但不要展示“好感度”“亲密度”“恋爱等级”等数值。

- 先做静态页面和 P0 骨架，不一次性做完整平台。
- 优先验证品牌表达和用户是否觉得“被理解”。
- 所有界面和文案保持中文。
- 移动端优先。
- 开发完成后说明改了哪些文件、为什么改、如何本地测试、如何确认无报错。

短期方向：

1. 完成 Landing Page 的静态页面。
2. 后续再进入注册/登录。
3. 再进入用户档案、角色选择、聊天页、记忆系统。

P0 的核心闭环是：

> 用户注册 → 建档 → 选择角色 → 聊天 → 被理解 → 被记住 → 愿意下次回来

Landing Page 当前采用 4屏结构：

1. Hero 情绪入口
2. 痛点共鸣 + 生活场景
3. 产品价值 + 简短体验
4. 角色入口 + CTA

当前原则：
首页不做过长，不做 6-7 屏功能堆叠。
第一版 Landing Page 优先验证用户是否觉得“被理解”，并尽快引导用户进入体验。




## 8. 禁止事项

产品理解上禁止把知你做成：

- AI 男友
- 乙女游戏
- 恋爱养成
- 攻略游戏
- 心理咨询工具
- 心理治疗产品
- 普通 AI 聊天机器人
- 效率工具

功能上 MVP 阶段不要做：

- 多人聊天
- 社区
- 用户 UGC 广场
- 角色互相聊天
- 复杂目标管理
- 复杂习惯养成
- 任务闯关系统
- 好感度明示数值
- 恋爱等级
- 成人色情内容
- 重度游戏化
- 微信登录

AI 回复禁止：

- 直接否定用户
- 说“你想多了”
- 说“大家都这样”
- 频繁讲道理
- 过度鸡汤
- 过度暧昧
- 过度心理分析
- 做医疗诊断
- 鼓励用户用 AI 替代现实关系
- 使用爹味、控制、PUA、霸总、油腻调情语气

如果用户表达严重自伤、自杀、伤害他人等内容，AI 应提供支持性回应，并建议用户联系现实中可信任的人或专业机构。

## 9. 给未来开发任务的提醒

开发时先判断当前任务属于哪个阶段，不要越级扩功能。

如果任务是 UI：

- 优先保持深夜、安静、私密、留白、低刺激。
- 不要为了“好看”加入乙游、二次元、粉色、满屏人物海报。

如果任务是角色：

- 优先区分情绪价值，不要只换职业、头像和外貌。

如果任务是 AI：

- 先共情，再理解，再回应。
- 记忆要服务于“被记住”，不是为了收集数据。

如果任务是产品功能：

- 优先 P0 闭环。
- 不要擅自添加社区、恋爱养成、心理治疗、复杂游戏化。

最终判断标准：

> 这个改动是否让用户更明确地感到：我被理解、被记住、被陪伴？
