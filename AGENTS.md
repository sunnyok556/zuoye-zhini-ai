# AGENTS.md

# 知你（Zhini）项目 AI 协作规则

本文件用于约束 Codex、Claude Code、Cursor 等 AI 编程助手的工作方式。

进入项目后必须优先阅读本文件。

---

# 一、接管顺序（强制）

任何 AI 接管项目时，必须按以下顺序阅读：

1. AGENTS.md
2. PROJECT_CONTEXT.md
3. PROJECT_SKILL.md
4. PROJECT_PROGRESS.md

然后输出：

1. 项目理解
2. 当前阶段
3. 已完成内容
4. 当前任务目标
5. 风险点
6. 建议方案

禁止直接开始写代码。

---

# 二、项目原则

知你（Zhini）是：

面向高责任感女性的 AI 情绪陪伴产品。

核心价值：

* 被理解
* 被记住
* 被陪伴

禁止理解为：

* AI男友
* 乙女游戏
* 恋爱养成
* 心理咨询工具
* 普通聊天机器人

所有设计与开发必须遵守：

深夜
安静
私密
克制
温柔
安全感

---

# 三、页面开发原则

每次只开发一个页面或一个区块。

禁止：

* 同时重构多个页面
* 顺手优化其它页面
* 扩大修改范围

例如：

当前任务是第三屏。

则：

只能讨论和修改第三屏。

禁止同时修改：

* Hero
* 第二屏
* 第四屏
* Header
* Footer

除非明确收到指令。

---

# 四、封版规则（最高优先级）

凡是在 PROJECT_PROGRESS.md 中标记：

"已封版"

的页面。

视为：

最终版本。

禁止：

* 重做
* 重构
* 推翻设计
* 更换视觉方向
* 修改核心文案
* 修改信息架构

允许：

* Bug修复
* 响应式修复
* 明确授权的小幅优化

如果发现封版页面存在问题：

先说明问题。

等待确认。

禁止直接修改。

---

# 五、视觉设计规则

设计前必须：

先查看当前页面实际效果。

禁止：

只阅读代码后直接判断设计。

必须：

Desktop + Mobile 同时检查。

必须查看：

* 页面截图
* 浏览器预览
* DevTools

之后才能提出设计建议。

---

# 六、屏幕边界规则（重要）

开发 Landing Page 时：

每一屏必须拥有明确边界。

禁止出现：

上一屏内容进入下一屏。

例如：

开发第三屏时：

禁止：

第二屏收尾文案出现在第三屏截图中。

必须保证：

当前截图只展示当前屏核心内容。

---

# 七、设计修改规则

优先：

小步迭代。

禁止：

因为一个细节问题推翻整个设计。

默认顺序：

1. 调整间距
2. 调整层级
3. 调整尺寸
4. 调整布局

最后才允许：

重设计

除非明确要求。

---

# 八、视觉验收流程

完成设计后必须：

Desktop：

* 1440×900

Mobile：

* 390×844

分别截图。

然后说明：

1. 修改了哪些内容
2. 为什么修改
3. 改动文件
4. 是否影响其它页面

禁止只提供代码。

必须先提供效果图。

---

# 九、开发验收流程

页面完成后：

必须执行：

pnpm lint

然后：

pnpm build

确认通过。

说明：

* 是否通过
* 是否存在警告
* 是否存在风险

之后才能进入提交阶段。

---

# 十、Git 提交流程

每完成一个独立阶段：

必须执行：

Step 1

commit

格式：

feat: xxx

或

docs: xxx

Step 2

tag

格式：

v0.x-description

例如：

v0.3-section2-final

Step 3

push

同步 GitHub

Step 4

更新 PROJECT_PROGRESS.md

记录：

* 当前阶段
* commit
* tag
* 是否封版
